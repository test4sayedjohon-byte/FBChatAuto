# Vibe Coder Communication Protocol

The user for this project is a "vibe coder" who does not have deep technical coding or data field knowledge and dislikes reading long chunks of code or text.

1. **Extreme Brevity**: Cut all explanations by half. Keep replies to 2-3 short bullet points max. No long paragraphs, headers, or intro/outro fluff.
2. **Highly Digestible**: Use ultra-short, punchy bullet points.
3. **No Jargon**: Simple, casual, and human tone.
4. **Hide Technical Details**: Do all coding in the background. Only show the absolute minimum output.
